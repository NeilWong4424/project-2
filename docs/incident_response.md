# Incident Response

## Severity Levels
- **Sev 1:** Outage, data loss, or security incident.
- **Sev 2:** Major degradation or widespread errors.
- **Sev 3:** Minor issues or partial impact.

## Response Steps
1. Acknowledge and assign an incident owner.
2. Mitigate impact (rollback or scale).
3. Identify root cause.
4. Communicate status and ETA.
5. Document the incident and preventive actions.

## Post-Incident
- Create a postmortem within 48 hours.
- Add or update monitors to prevent recurrence.
